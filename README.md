Project Reflection

What was the problem you were solving in the projects for this course?
I was solving the problem of efficiently loading, storing, and searching course data using different data 
structures. Each project focused on comparing how structure choice affects performance and usability.

How did you approach the problem?
I broke the tasks into clear steps: loading data, validating input, storing objects, and retrieving results—while 
considering how each data structure impacts speed and organization.

How did you overcome roadblocks during the activities or project?
I troubleshot issues by isolating small pieces of code, using print statements, reviewing pseudocode, and revisiting the 
logic behind each data structure until the program behaved correctly.

How has your work on this project expanded your approach to designing software?
It taught me to plan more intentionally, think about efficiency early, and design programs with clear data flow and 
structure instead of jumping straight into coding.

How has your work on this project evolved the way you write maintainable, readable, and adaptable programs?
I became more consistent with formatting, naming, and modular design, making my code easier to update, debug, and 
extend for future use.
